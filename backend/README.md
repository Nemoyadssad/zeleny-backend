# Бэкенд курса «Зелёный»

Небольшой сервер на Node.js, который закрывает то, что нельзя делать на одном статичном сайте:

- **Аккаунты** — регистрация и вход по e-mail и паролю.
- **Серверная проверка доступа** — оплачен курс или нет решает сервер, а не браузер. Подделать в DevTools нельзя.
- **Вебхук оплаты** — платёжная система сама сообщает серверу об успешной оплате, и доступ открывается.
- **ИИ-объяснения** — запрос к нейросети идёт через сервер, ключ API не виден в коде сайта.
- **Статистика** — отчёт по регистрациям и оплатам.

## Что нужно
- Node.js версии 18 или новее (`node -v` для проверки).

## Запуск на своём компьютере (для теста)
```bash
cd backend
npm install                # установить зависимости
cp .env.example .env       # создать файл настроек (на Windows: copy .env.example .env)
# открыть .env и заполнить значения (минимум JWT_SECRET и ADMIN_TOKEN)
npm start                  # запустить сервер
```
Сервер поднимется на `http://localhost:8080`. Проверка: открой в браузере `http://localhost:8080/api/stats?token=ТВОЙ_ADMIN_TOKEN` — должен вернуться JSON со статистикой.

## Переменные окружения (.env)
| Переменная | Зачем |
|---|---|
| `JWT_SECRET` | секрет для токенов входа — длинная случайная строка |
| `ANTHROPIC_API_KEY` | ключ для ИИ-объяснений |
| `PAYMENT_WEBHOOK_SECRET` | секрет для проверки подписи вебхука оплаты |
| `ADMIN_TOKEN` | секрет для доступа к статистике и ручной выдаче доступа |
| `ALLOWED_ORIGIN` | домен сайта, которому разрешены запросы (напр. `https://zeleny.ru`) |
| `AI_MODEL` | модель ИИ (по умолчанию дешёвая haiku) |
| `SERVE_STATIC` | путь к папке сайта, если хотите отдавать сайт с этого же сервера |

## Эндпоинты (что умеет сервер)
| Метод и путь | Назначение |
|---|---|
| `POST /api/register` | регистрация: `{name, email, password}` → возвращает токен |
| `POST /api/login` | вход: `{email, password}` → токен |
| `GET /api/me` | статус доступа (нужен заголовок `Authorization: Bearer ТОКЕН`) |
| `POST /api/payment/webhook` | вызывает платёжка после оплаты: `{email, plan}` |
| `POST /api/admin/grant` | выдать доступ вручную (заголовок `x-admin-token`) |
| `GET /api/stats?token=ADMIN_TOKEN` | отчёт: регистрации и оплаты |
| `POST /api/ai/explain` | ИИ-объяснение вопроса (нужен токен и оплата) |

## Тарифы и срок доступа
В вебхуке поле `plan`:
- `month` — доступ на 30 дней (поле `paid_until` ставится автоматически);
- `full` (или любое другое) — доступ навсегда.

## Развёртывание в интернете
Подойдёт любой хостинг Node.js: **Render**, **Railway**, **Fly.io** или обычный VPS.

Общий порядок:
1. Залейте папку `backend` в репозиторий (без `node_modules` и `.env` — они в `.gitignore`).
2. На хостинге создайте сервис из этого репозитория, команда запуска `npm start`.
3. Пропишите переменные окружения из `.env` в настройках хостинга.
4. Получите адрес сервиса, например `https://api.zeleny.ru`.
5. В коде сайта (`index.html`) укажите этот адрес в настройке `API_BASE` — это следующий шаг.
6. В кабинете платёжной системы укажите адрес вебхука: `https://api.zeleny.ru/api/payment/webhook`.

## Важно про вебхук оплаты
Каждая платёжная система шлёт вебхук в своём формате и со своим способом подписи. В `server.js` сделана базовая проверка подписи как HMAC-SHA256 от тела запроса с заголовком `x-signature`. Под вашего провайдера (ЮKassa, CloudPayments, Robokassa и т.д.) нужно поправить:
- имя заголовка с подписью;
- способ вычисления подписи (у некоторых это не HMAC всего тела);
- из каких полей вебхука брать e-mail покупателя и тариф.

Это место в коде помечено комментарием. Если пришлёшь, какой провайдер используешь, подскажу точные правки.
